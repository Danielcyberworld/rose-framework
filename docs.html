First Things First

You can view full code segments on the examples page

Rose is designed to be accessible by any designer. You've heard of SMACSS, BEM, or Atomic CSS, so Rose is designed to pick the best of each. With a lightweight file-size and incredible user-customizability, Rose is an easy solution to responsive development .

Rose is split into 3 files: theme, base, and modules. In the theme stylesheet, you can edit the base and accent colors used for the default .accent and .base colors. These classes are used throughout, and are set with SCSS. If you would like to have a unified theme for your site, make sure to change the $base and $accent variables with a SCSS compiler. You can also add your own colors to use as classes. Just add a variable of your choice, say, $var, and then add .var-bg to set the background-color, and .var to set the text-color. The font variables are also set here.

In the base stylesheet, the actual fonts are set, as well as some standardization similar to but not based on normalize.css. In addittion, media queries are used to set font sizes and weights for different sizes. You can edit those in this stylesheet, as well as basic styling for most HTML tags.

Finally, the modules stylesheet sets up all the modules and setup classes that Rose uses. If you need to, you can edit those here. The stylesheet is divided into 5 media queries: Monitor, for a large display, Desktop, for an average display, Laptop, for a small display, Tablet, for tablet displays, and Mobile, for mobile displays. The modules are pretty well commented, so just look for the /* LABEL */ comments in the code. You'll find the modules neatly arranged underneath.

DTS System

Rose is based on the DTS System, aka the Display-Trait_Size system.

Rose includes four displays: lg, md, sm, and xs. The desktop and monitor displays use lg, laptops use md, tablets use sm, and mobile devices use xs.

The trait is a common style, such as margin, padding, margin-top, and so on.

Size comes in four varietes, xs, sm, md, and lg. Unlike the display, these are not fixed for the size of the screen, but rather relate to the size relative to each other. For best practice, one of each styles should be applied whenever at least one is used. That doesn't make sense? Yeah, here's what that means: if you use, say lg-margin-top_xs, its best to specify md-, sm-, and xs-margin-top_. So, if you wanted a div to have a medium amount of left margin on the smallest display, you can add the xs-margin-left_md class.

Semantic Support

HTML5 was created with the intention to add meaning behind the markup. Rose and its modules are written to support and supplement semantic markup.

Unlike other frameworks, which rely on a sea of divs, Rose is designed so that you can take full advantage of the new semantc tags to allow your sites to not only look great, but to make semantic sense as well.

The Rose Grid System

Rose ships with a super flexible grid system. Rose pages consist of grids and units. There are two types of grids (like Bootstrap): grid-full and grid. The .grid-full grid takes up 100% of its parent element, and is therefore the full width of the page on all devices. The .grid grid takes up 85% of its parent container on lg and md devices (width a max-width of 1124px) and the full width on sm and xs devices.

Within each grid are elements. An element can be either a unit, or a regular element, styled with rose modules. Rose units take up fractions of their parent grid. Units ship with base 16, 12, 8, 6, 4, 3, 2, and 1 fractions (simplified fractions are supported). By default, these units have margin, but Rose also ships with unit-flat, a unit with no margin. For a unit, the style is (screen-size)-numerator-denomenator. For example, an unit taking up 1/3 of its parent grid on a small device would be .sm-1-3. Its that simple. Units are nestable as well. For a unit-flat (a no-margin unit), use (screen-size)-flat-numerator-denominator. Finally, the units (flat and regular) ship with offset, up to the nearest max. Lets look at some examples.

The Rose Navigation System

Rose Navigation is built to be simple and responsive, with a large amount of customization. For starters, the navigation is a semantic nav. Inside of the nav are up to three sections. Each section must be dictated either nav-center, nav-left, or nav-right. The nav is split into thirds and each section is assigned a third.

A unique feature of the Rose navigation bar is the navbar initial class. If you choose to do so, you can customize the navigation bar to change when it is past a certain point on the page. This point is marked with the id of top. For this page, the id="top" section is marked at the very top of the page, so the navigation changes immediately upon scrolling past the top of the banner image. To access this, within each media query of the modules stylesheet, at the top are the nav and .navbar-initial selectors. Modify .navbar-initial to alter the styles before passing the scroll point, and then set the default styles in the nav selector. If you do not wish to use this feature, set your styles in the nav selector, and copy them to the .navbar-initial selector for each media query. Rose navigation is also designed with responsiveness in mind, so the navigation bars collapse into mobile ready versions.

Covers, Cover-image, Cover-small, Cover-small-image

Covers are the pieces used in the full image background design trend. A .cover is a full-height container of content that can have the background set to any color, and contains a large amount of content or space on the site. On most screen sizes these are 100VH.

The .cover-image is section like .cover, but you can set a background-image extremely easily with the data-src attribute. Simply add the data-src attribute and set it equal to the src of the background image.

A .cover-small is roughly 50 to 60Vh.

A .cover-small-image is the same size as a .cover-small, but with an image (set with the data-src attribute). One of these is used at the top of the page.

All Rose modules

Here is the full list of modules, with documentation on each. You should note that there are two built in fallbacks. First, for md and lg screens, the md- styles are the default fallback. For xs and sm devices, the xs- styles are the default fallback. There fore, you may only need to use md- and xs- styles if your page layout is fairly simple.
